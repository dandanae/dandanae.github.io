null, undefined, undeclared, NaN

이 글에 대하여

자바스크립트에서 값이 “없다/정의되지 않았다/숫자가 아니다”를 표현하는 방법은 각각 달라요.

#Frontend#JavaScript#null#undefined#NaN

null

  • 값이 없다는 것을 의도적으로 표현하는 값이에요.
  • 개발자가 변수에 직접 할당해 “비어있음”을 명시할 때 사용해요.
  • 타입은 object로 나오는데, 이는 자바스크립트 초창기 버그이자 역사적 유물이에요.
1
let value = null; 

2
console.log(value); // null

undefined

  • 변수는 선언됐지만 값이 할당되지 않은 상태를 의미해요.
  • 자바스크립트 엔진이 기본으로 부여하는 값이에요.
  • return이 없는 함수도 자동으로 undefined를 반환해요.
1
let value;

2
console.log(value); // undefined

undeclared

  • 아예 선언되지 않은 변수를 접근할 때 발생해요.
  • 사용 시 ReferenceError가 발생해요.
1
console.log(notDefined); 

2
// ReferenceError: notDefined is not defined

NaN (Not a Number)

  • 이름 그대로 숫자가 아님을 나타내는 특별한 숫자 값이에요.
  • 숫자 연산이 실패했을 때 반환돼요.
  • typeof NaN은 number예요.
    - NaN === NaN은 false라서, 확인할 땐 Number.isNaN()을 써야 해요.
