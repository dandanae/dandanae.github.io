JavaScript 비동기 처리 방법

이 글에 대하여

자바스크립트는 싱글 스레드 언어이지만, 비동기 처리를 통해 동시에 여러 작업을 처리하는 것처럼 동작해요. 대표적인 방식은 콜백, 프로미스(Promise), async/await이에요.

#Frontend#JavaScript#Promise#async/await

콜백 (Callback)

  • 특정 작업이 끝난 뒤 실행할 함수를 인자로 전달하는 방식이에요.
  • 콜백 중첩이 심해지면 가독성이 떨어져요. (콜백 지옥)
1
function fetchData(callback) {

2
  setTimeout(() => {

3
    callback("서버에서 받은 데이터");

4
  }, 1000);

5
}

6


7
fetchData((data) => {

8
  console.log(data);

9
});

프로미스 (Promise)

  • 비동기 작업의 성공/실패 상태를 다루는 객체예요.
  • then / catch 체이닝으로 흐름을 깔끔하게 표현할 수 있어요.
1
const fetchData = () => {

2
  return new Promise((resolve, reject) => {

3
    setTimeout(() => resolve("데이터 도착"), 1000);

4
  });

5
};

6


7
fetchData()

8
  .then((res) => console.log(res))

9
  .catch((err) => console.error(err));

async / await

  • 프로미스를 더 직관적으로 사용할 수 있게 만든 문법이에요.
  • await 키워드로 동기 코드처럼 작성할 수 있어요.
  • 오늘날에는 대부분 async/await이 표준처럼 쓰이고 있어요.
1
const fetchData = () => {

2
  return new Promise((resolve) => {

3
    setTimeout(() => resolve("데이터 도착"), 1000);

4
  });

5
};

6


7
async function getData() {

8
  const result = await fetchData();

9
  console.log(result);

10
}
