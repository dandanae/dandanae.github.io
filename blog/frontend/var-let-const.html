var, let, const의 차이점

이 글에 대하여

세 키워드는 모두 변수를 만들지만, 스코프·재할당 여부·호이스팅 동작에서 차이가 있어요.

#Frontend#JavaScript#var#let#const

var

  • 함수 스코프(Function Scope) 를 가져서, 블록 안에서 선언해도 함수 전체에서 접근 가능해요.
  • 중복 선언이 가능하지만, 실수로 변수를 덮어쓸 수 있어 위험해요.
  • 호이스팅 시 undefined로 초기화돼서, 선언 이전에 접근하면 undefined가 출력돼요.
1
console.log(a); // undefined

2
var a = 10;

3
console.log(a); // 10

let

  • 블록 스코프(Block Scope) 로, {} 블록 안에서만 유효해요.
  • 중복 선언이 불가능해요.
  • 호이스팅은 되지만 TDZ(Temporal Dead Zone) 때문에 초기화 전 접근 시 에러가 발생해요.
1
console.log(b); // ReferenceError

2
let b = 20;

const

  • 블록 스코프(Block Scope) 로, {} 블록 안에서만 유효해요.
  • 중복 선언이 불가능해요.
  • 상수로서, 재할당이 불가능해요.
  • 단, 객체나 배열 같은 참조형 데이터는 내부 값은 수정 가능해요.
1
const c = 30;

2
c = 40; // ❌ TypeError

3


4
const obj = { name: "홍길동" };

5
obj.name = "수정 가능"; // ✅ 내부 속성 수정은 가능
