Hash / Map

이 글에 대하여

해시는 “키를 만드는 원리”, 맵은 “키-값을 저장하는 구조”라고 보면 이해가 쉬워요.

#

해시(Hash)란?

  • 데이터를 빠르게 검색·저장하기 위해 사용하는 함수적 개념이에요.
  • 입력값 → 고정된 크기의 해시 값(Hash Value) 으로 변환해요.
  • 해시 함수(Hash Function)를 사용해서 데이터를 인덱싱해요.

특징

  • 빠른 검색이 가능해요. (O(1) 기대 시간 복잡도)
  • 충돌 *이 가능해요.
  • 보통 해시 테이블(Hash Table) 이라는 구조로 구현해요.

맵(Map)

  • 데이터를 키-값 쌍(key-value pair) 으로 저장하는 자료구조예요.
  • 키를 기준으로 값을 저장/검색/삭제할 수 있어요.
  • 자바스크립트의 Map 객체도 이 개념을 따라요.
  • 실무에서는 Map이 더 유연하고 강력하기 때문에 권장돼요.

특징

  • 키는 어떤 타입(객체, 함수 등)도 가능해요.
  • 삽입 순서를 기억해요.
  • 해시 테이블로 구현되는 경우가 많아 빠른 접근이 가능해요.
