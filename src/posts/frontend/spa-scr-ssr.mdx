# SPA (Single Page Application)
**하나의 HTML 파일**만 로드하고, 이후에는 **JavaScript로 동적으로 화면을 교체**하는 방식이에요.
라우팅도 클라이언트에서 처리하고, 필요한 데이터만 API로 받아와 갱신해요.

## 특징
- 첫 로딩은 느릴 수 있지만, 이후엔 빠르고 부드럽게 동작해요.
- 페이지 전환 시 새로고침이 없어서 앱(App) 같은 경험을 줘요.
- SEO가 불리할 수 있어요
  → 보완책으로 SSR, SSG를 같이 쓰기도 해요.\

# CSR (Client-Side Rendering)
브라우저가 서버에서 **빈 HTML + JS 번들**을 받아오고, 그 **JS가 실행되면서 화면을 그리는 방식**이에요.
대부분의 SPA 프레임워크(React, Vue 등)는 CSR을 기반으로 동작해요.

## 특징
- 서버는 단순히 **정적 리소스(JS, CSS)** 만 제공해요.
- 초기 로딩이 상대적으로 느려요. (자바스크립트 다운로드 + 실행 후 렌더링).
- 이후 화면 전환은 매우 빨라요.

# SSR (Server-Side Rendering)
사용자가 페이지를 요청하면, 서버가 **완성된 HTML을 만들어서 바로 전달**해요.
브라우저는 이를 즉시 렌더링하고, 이후 JS가 실행되면서 상호작용이 가능해져요.

## 특징
- 초기 로딩 속도가 빠르고, SEO에 유리해요.
- 하지만 서버 부하가 크고, 요청마다 서버에서 HTML을 다시 만들어야 해서 성능 이슈가 있을 수 있어요.
- Next.js(React) 같은 프레임워크에서 많이 사용돼요.

# 비교 요약
| 구분          | SPA          | CSR          | SSR                        |
| ------------- | ------------ | ------------ | -------------------------- |
| **초기 로딩** | 느림         | 느림         | 빠름                       |
| **화면 전환** | 매우 빠름    | 매우 빠름    | 새 요청 필요 (상대적 느림) |
| **SEO**       | 불리         | 불리         | 유리                       |
| **서버 부하** | 낮음         | 낮음         | 높음                       |
