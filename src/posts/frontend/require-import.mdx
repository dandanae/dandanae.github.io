# require (CommonJS)
- **Node.js에서 기본적으로 사용**되는 모듈 시스템이에요.
- 런타임(코드 실행 시점)에 모듈을 불러와요.
- 동기 방식으로 동작하기 때문에 파일 시스템 기반(Node.js 환경)에 적합해요.

```js
// math.js
module.exports.add = (a, b) => a + b;

// app.js
const math = require('./math');
console.log(math.add(2, 3)); // 5
```

# import (ES Module, ESM)
- **ECMAScript 표준 모듈 시스템**이에요.
- 브라우저와 Node.js(ESM 지원 모드) 모두에서 사용 가능해요.
- 컴파일/파싱 단계에서 정적으로 분석돼서, **트리 쉐이킹(Tree-shaking)** 같은 최적화가 가능해요.
- 비동기 로딩도 지원해요(import() 동적 임포트).

```js
// math.js
export const add = (a, b) => a + b;

// app.js
import { add } from './math.js';
console.log(add(2, 3)); // 5
```

# 주요 차이점
| 구분      | require                  | import                      |
| --------- | ------------------------ | --------------------------- |
| 소속      | CommonJS                 | ES Module(ES6)              |
| 로딩 시점 | 런타임(실행 중)          | 컴파일/파싱 단계            |
| 동작 방식 | 동기                     | 정적(기본), 동적도 가능     |
| 사용 환경 | Node.js(기본)            | 브라우저, Node.js(ESM 모드) |
| 최적화    | 불가                     | 트리 쉐이킹 가능            |
| 문법      | `const a = require('a')` | `import a from 'a'`         |
