# 실행 컨텍스트란?
자바스크립트 엔진이 코드를 실행할 때 **필요한 정보들을 모아둔 객체**예요.
여기에는 **변수, 함수 선언, 스코프 체인, this 바인딩 등**이 포함돼요.
코드 실행 과정에서 **무엇을 참조해야 할지 결정하는 기준**이 돼요.

# 실행 컨텍스트 종류
## 전역 컨텍스트(Global Context)
- 코드가 처음 실행되면 만들어지는 컨텍스트예요.
- 전역에 선언된 변수/함수, `window`(브라우저)나 `global`(Node.js) 같은 전역 객체가 포함돼요.
- 프로그램이 끝날 때까지 유지돼요.

## 함수 컨텍스트(Function Context)
- 함수가 호출될 때마다 생성돼요.
- 함수 내부의 변수, 매개변수, arguments, this 등이 담겨요.
- 함수 실행이 끝나면 파괴돼요(단, [클로저](/blog/frontend/closure)처럼 참조가 남으면 유지돼요).

# 실행 컨텍스트 구성 요소
## 변수 환경(Variable Environment)
- `var`로 선언된 변수, 함수 선언문을 저장해요.

## 렉시컬 환경(Lexical Environment)
- `let`, `const` 변수, 스코프 체인 정보를 저장해요.

## this 바인딩
- 실행 시점에 `this`가 무엇을 가리킬지를 결정해요.

# 예시
```js
var x = 1;

function outer() {
  var y = 2;
  
  function inner() {
    var z = 3;
    console.log(x, y, z);
  }

  inner();
}

outer();
```

출력
```plainText
1 2 3
```

1. **전역 컨텍스트 생성**
  자바스크립트가 실행되면 **제일 먼저 전역 실행 컨텍스트(Global Execution Context)** 가 생겨요.
  여기에는 **`x = 1`, `outer` 함수 선언이 등록**돼요.

2. **`outer()`가 호출되면, `outer` 실행 컨텍스트 생성**
  새로운 함수 실행 컨텍스트가 만들어져요.
  여기에는 **`y = 2`, `inner` 함수 선언이 저장**돼요.

3. **`inner()`가 호출되면, `inner` 실행 컨텍스트 생성**
  또 하나의 실행 컨텍스트가 생겨요.
  여기에는 **`z = 3`이 저장**돼요.

4. **`console.log(x, y, z)` 실행**
  자바스크립트는 변수를 찾을 때 **스코프 체인**을 따라가요.
  탐색 순서는 **항상 현재 실행 컨텍스트(=inner)에서 시작 → 바깥(outer) → 전역 순서**예요.
    - `x` → `inner`에 없음 → `outer`에 없음 → `전역`에서 발견
    - `y` → `inner`에 없음 → `outer`에서 발견
    - `z` → `inner`에서 바로 발견
