# 쿠키(Cookie)
- **서버와 클라이언트 간 상태를 유지**하기 위해 사용하고, 요청 시 **자동으로 서버에 전송**돼요.
- 도메인별로 **브라우저에 저장**하며, 약 **4KB 내외**의 용량을 가지고 있어요.
- 만료 기한을 설정할 수 있어요.
- **로그인 세션, 사용자 식별** 등에 주로 사용해요.
- 보안 위험(세션 하이재킹 등)이 있어 `HttpOnly`, `Secure` 옵션을 자주 사용해요.

# 로컬 스토리지(Local Storage)
- 사용자가 지우지 않는 한 **브라우저에 영구적으로 유지**돼요.
- 도메인당 약 **5MB**의 용량을 가지고 있어요.
- 기본적으로 **문자열만 저장 가능**하지만, JSON을 활용하면 객체도 저장할 수 있어요.
- 요청 시 서버에 자동으로 전송되지 않는, **순수 클라이언트 측 저장**이에요.

# 세션 스토리지(Session Storage)
- **탭 단위 세션을 유지**하기 위해 사용해요.
- 도메인당 약 **5MB**의 용량을 가지고 있어요.
- **탭을 닫으면 사라지지만, 새로고침 시에는 유지**돼요.
- 기본적으로 **문자열만 저장 가능**하지만, JSON으로 객체를 저장할 수 있어요.
- 탭/창 단위로 격리돼서 **다른 탭에서는 접근할 수 없어요.**

# 인덱스드DB(IndexedDB)
- 브라우저 내 **대용량 데이터베이스 저장소**예요.
- 브라우저마다 다르지만 **수백 MB 이상**의 용량을 가지고 있어요.
- 만료 기한이 없어요.
- **구조화된 데이터(객체, 파일, 블롭 등)** 를 저장할 수 있어요.
- 트랜잭션을 지원하고 비동기 API를 제공해요.
- **PWA(Progressive Web App)** 에서 오프라인 데이터 저장 시 자주 활용돼요.
