# 콜백 (Callback)
- 특정 작업이 끝난 뒤 실행할 함수를 **인자로 전달**하는 방식이에요.
- 콜백 중첩이 심해지면 가독성이 떨어져요. (콜백 지옥)

```js
function fetchData(callback) {
  setTimeout(() => {
    callback("서버에서 받은 데이터");
  }, 1000);
}

fetchData((data) => {
  console.log(data);
});
```

# 프로미스 (Promise)
- 비동기 작업의 **성공/실패 상태**를 다루는 객체예요.
- `then` / `catch` 체이닝으로 흐름을 깔끔하게 표현할 수 있어요.

```js
const fetchData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => resolve("데이터 도착"), 1000);
  });
};

fetchData()
  .then((res) => console.log(res))
  .catch((err) => console.error(err));
```

# async / await
- 프로미스를 더 직관적으로 사용할 수 있게 만든 문법이에요.
- `await` 키워드로 **동기 코드처럼 작성**할 수 있어요.
- 오늘날에는 대부분 **async/await**이 표준처럼 쓰이고 있어요.

```js
const fetchData = () => {
  return new Promise((resolve) => {
    setTimeout(() => resolve("데이터 도착"), 1000);
  });
};

async function getData() {
  const result = await fetchData();
  console.log(result);
}

getData();
```
